## 💻 実装のこだわり・技術的ハイライト



静的サイトだからこそ、「コードの保守性」と「ユーザー体験（UX）」に徹底的にこだわりました。



### 1. 拡張性と保守性を意識したCSS設計

多ページ展開を見据え、**FLOCSS**（Foundation, Layout, Object）をベースにしたディレクトリ構成とクラス命名規則を採用しました。



| カテゴリ | 実装詳細 |

| :--- | :--- |

| **変数の活用** | 色、フォント、共通の余白を `:root` で変数管理し、一括変更に強い設計にしました。 |

| **コンポーネント化** | ボタン (`.c-button`)、カード (`.c-card`)、タイトル (`.c-title`) を共通パーツとして切り出し、再利用性を高めています。 |

| **レイアウト分離** | ヘッダー・フッター等の大枠 (`layout/`) と、ページ固有のスタイル (`pages/`) を明確に分離し、修正時の影響範囲を限定しました。 |



### 2. モダンなレスポンシブ・UI実装

スマホファーストで記述し、PC表示（`min-width: 769px`）へ拡張する形で実装しています。



* **`clamp()` 関数の活用**:

    * `font-size: clamp(14px, 2vw, 16px);` 等を使用し、デバイス幅に応じて滑らかに文字サイズや余白が可変する「リキッドレイアウト」を実現しました。

* **CSS Grid / Flexbox**:

    * 商品一覧やニュース記事の配置に CSS Grid を採用。デバイスごとのカラム落ちやレイアウト変更を柔軟に実装しました。



### 3. SEOとアクセシビリティへの配慮

* **セマンティックHTML**:

    * `<header>`, `<main>`, `<article>`, `<section>` などを適切に使用し、マシンリーダブルな文書構造を意識しました。

* **画像の最適化**:

    * `<picture>` タグを使用し、WebP形式とJPEG形式を出し分けることで、画質を保ちつつ読み込み速度を高速化させました。

* **フォームのユーザビリティ**:

    * `input` タグの `type="email"` `type="tel"` や `autocomplete` 属性を適切に設定し、モバイルユーザーの入力ストレスを軽減しました。



## 🤖 開発スタイル・AI活用



コーディングの効率化と技術習得のために、生成AI（Gemini）を「ペアプログラミングのパートナー」として活用しました。



* **HTML/CSS**:

    * 自分で設計・コーディングを行い、命名規則の確認などにAIを利用。

* **JavaScript**:

    * ハンバーガーメニューやスクロール連動アニメーションなど、実現したい挙動をAIに指示。生成されたコードをそのまま使うのではなく、プロジェクトの設計に合わせて調整・リファクタリングして導入しました。
